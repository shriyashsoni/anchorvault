# Mathematics & Yield Proofs

To avoid floating-point non-determinism across Stellar's distributed consensus nodes, the Soroban runtime executes calculations strictly in fixed-point integer spaces. AnchorVault implements safe checked arithmetic scaled to **7 decimal places (10^7 scaling factor)** to preserve absolute precision.

---

## 1. Dynamic LP Share Minting Formula

When a liquidity provider deposits capital `D`, the dynamic allocation of LP shares `S_minted` is calculated mathematically based on existing total shares `S_total` and total pooled reserve assets `R_total`:

> **S_minted = D * (S_total / R_total)**

### Boundary Condition:

> **If S_total = 0, then S_minted = D**

### Rounding Attack Mitigation:
To protect existing liquidity providers against fractional share draining (inflation attacks), the contract strictly **rounds down** on deposits using the `checked_div` operator. Any decimal fractional remainder is permanently committed to the vault reserve, boosting backing value for all existing providers.

---

## 2. Dynamic Share Redemption Payout Formula

When a liquidity provider burns `S_redeem` shares to withdraw their capital, the payout assets `A_payout` are determined proportionally:

> **A_payout = S_redeem * (R_total / S_total)**

### Rounding Safeguards:
During withdrawal burning operations, decimal calculations strictly **round up** (by adding 1 to the integer division if a remainder exists). This acts as a mathematical safeguard, guaranteeing that the vault never distributes more backing stablecoins than the LP's shares mathematically represent.

---

## 3. High-Precision Integer SafeMath

All mathematical additions, subtractions, and allocations wrap securely in verified Rust SafeMath blocks. Floating point operations are strictly banned on the Soroban runtime to avoid minor variations across distributed ledger consensus checks.

```rust
// Verified Rust Checked Arithmetic wrap in CoreVault
let next_shares = current_shares
    .checked_add(minted_amount)
    .ok_or(VaultError::ArithmeticOverflow)?;
```

---

## 4. Yield Compounding Model

As registered Cash-In/Cash-Out (CICO) anchors generate corridor commissions, they programmatically invoke `distribute_yield`. This directly inflates `R_total` without increasing `S_total`:

> **Yield Allocation Rate = (R_total + ΔR) / S_total**

As a result, each existing LP share represents a progressively larger amount of USDC stablecoin reserves over time, driving organic compound yield distribution.
