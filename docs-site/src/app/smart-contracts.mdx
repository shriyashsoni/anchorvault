# Deployed Smart Contracts

AnchorVault utilizes a highly optimized, modular smart contract suite consisting of two main components: **CoreVault Engine** and **AnchorRegistry**. Dividing these responsibilities optimizes WASM sizes, scales gas consumption, and separates staking security pools from daily remittance pipelines.

---

## 1. CoreVault Engine

The **CoreVault Engine** handles liquidity reserves, LP deposits, token share allocations, and organic interest/commission payouts.

*   **Module Type:** Vault Module
*   **Testnet Address:** `CCU3RFCKEG2OIQZMGY6C2UUQFCCN6TJDVMPNRR3D6FKRZAJGQ3EIPKJK`

### Storage Architecture (`StorageType::Instance`)

To persist critical protocol values securely, CoreVault uses Soroban's **Instance Storage**:

| Storage Key | Data Type | Description |
| :--- | :--- | :--- |
| `Admin` | `Address` | Deployed administrator authorized to change commission fees and register oracles. |
| `AssetToken` | `Address` | Standard Stellar Asset Contract (SAC) representing the underlying backing token (USDC/EURC). |
| `TotalShares` | `i128` | Dynamically tracks all active liquidity provider shares minted to date. |
| `TotalDeposits` | `i128` | Balance ledger representing total physical reserves currently locked in the vault. |

### Public Contract Functions

```rust
// Binds deployer credentials and sets primary backing stablecoin addresses
pub fn initialize(env: Env, admin: Address, asset: Address);

// Transfers USDC capital from user to vault reserves, minting proportional LP shares
pub fn deposit(env: Env, user: Address, amount: i128) -> i128;

// Redeems and burns LP shares, returning equivalent USDC stablecoins plus accrued yield
pub fn withdraw(env: Env, user: Address, shares: i128) -> i128;

// Triggers dynamic yield injection generated by active cross-border corridors
pub fn distribute_yield(env: Env, amount: i128);
```

## 2. AnchorRegistry

The **AnchorRegistry** manages cross-borderCash-In/Cash-Out (CICO) anchor credentials, locks reputation stakes, registers credit limits, and processes automated slashing.

*   **Module Type:** Registry Module
*   **Testnet Address:** `CAWO6A52CISR4JITVFVN4NDDCSJA3MI5N6XCBN5XW2AE4JU3I4NHAUGJ`

### Public Contract Functions

```rust
// Registers a regulated Cash-Out anchor, locking their reputation bond
pub fn register_anchor(env: Env, anchor: Address, bond: i128);

// Opens a 72-hour interactive challenge window against an underperforming gateway
pub fn flag_dispute(env: Env, merchant: Address, proof: BytesN<32>);

// Allows anchors to resolve flags by presenting cryptographic proof of payout
pub fn resolve_dispute(env: Env, dispute_id: u64, receipt: BytesN<32>);

// programmatically slashes staked bonds to refund merchants in case of default
pub fn liquidate_anchor(env: Env, dispute_id: u64);
```
