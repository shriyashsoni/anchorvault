# SDK & Client Integration

Interact programmatically with AnchorVault smart contracts using modern Web3 client tools. This guide provides production-grade codeblocks for JavaScript/TypeScript environments and native Rust cross-contract clients.

---

## 1. Typescript SDK Integrations

### Wallet Adaptors (Freighter Chrome Extension)

Detect, connect, and request transaction signatures securely using the official `@stellar/freighter-api` package:

```typescript
import { isConnected, signTransaction } from "@stellar/freighter-api";

/**
 * Requests Freighter to sign a Soroban transaction XDR
 * @param txXdr The built transaction XDR string
 */
export async function signWithFreighter(txXdr: string): Promise<string> {
  if (await isConnected()) {
    try {
      const signedXdr = await signTransaction(txXdr, {
        network: "TESTNET",
        networkPassphrase: "Test SDF Network ; September 2015",
      });
      return signedXdr;
    } catch (err) {
      throw new Error(`User rejected signature: ${err}`);
    }
  }
  throw new Error("Freighter browser extension not found. Please install the extension.");
}
```

### Complete Invocation Workflow

Build, simulate, and submit a secure LP deposit transaction using the dynamic `@stellar/stellar-sdk` library. Remember to scale decimal numbers to **7 decimal places (10^7)**:

```typescript
import { Contract, rpc, TransactionBuilder, nativeToScVal } from "@stellar/stellar-sdk";

// Initialize Soroban Testnet RPC Server
const server = new rpc.Server("https://soroban-testnet.stellar.org");
const contractId = "CCU3RFCKEG2OIQZMGY6C2UUQFCCN6TJDVMPNRR3D6FKRZAJGQ3EIPKJK";

/**
 * Builds and dispatches a deposit transaction for LP reserves
 * @param senderAddress The public G-address of the user
 * @param amountDecimal The decimal amount of USDC to deposit (e.g. 150.50)
 */
export async function executeDeposit(senderAddress: string, amountDecimal: number): Promise<string> {
  // 1. Fetch active account sequence ledger indices
  const sourceAccount = await server.getLedgerAccount(senderAddress);
  
  // 2. Scale up decimals to high-precision BigInt integers
  const amountInteger = BigInt(Math.floor(amountDecimal * 10 ** 7)); 

  // 3. Construct Contract and build transaction pipeline
  const vaultContract = new Contract(contractId);
  const tx = new TransactionBuilder(sourceAccount, { fee: "100000" }) // Set standard gas constraints
    .addOperation(
      vaultContract.call(
        "deposit",
        nativeToScVal(senderAddress, { type: "address" }),
        nativeToScVal(amountInteger, { type: "i128" })
      )
    )
    .setTimeout(30)
    .build();

  // Return base64 encoded XDR ready for Freighter signing
  return tx.toXDR();
}
```

## 2. Rust Cross-Contract Connections

To invoke the CoreVault Engine directly from other smart contracts on the Soroban runtime, import the WASM client using the `contractimport!` macro:

```rust
use soroban_sdk::{contract, contractimpl, Address, Env};

// Import client definitions dynamically from build artifacts
mod vault_contract {
    soroban_sdk::contractimport!(file = "target/wasm32-unknown-unknown/release/anchor_vault.wasm");
}

#[contract]
pub struct RemittanceGatewayConnector;

#[contractimpl]
impl RemittanceGatewayConnector {
    /// Interacts directly with the vault reserves to deposit LP stakes
    pub fn deposit_lp_capital(
        env: Env, 
        vault_addr: Address, 
        user: Address, 
        amount: i128
    ) -> i128 {
        let client = vault_contract::Client::new(&env, &vault_addr);
        
        // Execute cross-contract transaction, resolving dynamic sharing calculations
        client.deposit(&user, &amount)
    }
}
```
